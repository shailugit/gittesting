ASP.NET Important files & structure

  • Project File: Specify target application & framework, how application hosted, InProcess/OutProcess, Include NuGet references.
  • The default is InProcess where app host inside IIS, OutProcess is where app host in Krestel App Server
  • Program.cs:
  • Startup.cs: Two main methods Configure() & ConfigureServices()
  • launchsettings.json: Used for local development, it has Authentication, IIS, profiles app url kind of settings
  • appsettings.json: Used for publishing

    • Program.CS: Main Method
    public static void Main(string[] args) { CreateWebHostBuilder(args).Build().Run(); }
    public static IWebHostBuilder CreateWebHostBuilder(string[] args) => WebHost.CreateDefaultBuilder(args) .UseStartup();

    .NET Core run as a Console application, this method configure the app and start it, it's an entry point and call CreateWebHostBuilder(). CreateWebHostBuilder() return IWebHostBuilder, on this Build() method is called which builds a web host & Run() will run webapplication.
    WebHost.CreateDefaultBuilder() will create WebHost with default config files it also configure the Startup class. Its a static method of WebHost class and perform task like -Setting up webserver, Load host & config files, Config logging. In case of InProcess CreateDefaultBuilder call UseIIS() method and host app inside IIS.

    StartUp Class: Having two main methods

  • ConfigureServices(): Method to config services in .NET Core
    Services are register like services.AddMvc(); services.AddSingleton();
  • Configure(): Method to setup application request processing pipeline.
